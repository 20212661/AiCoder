## 一、项目概览

Aider 是一个功能完善的 **AI 结对编程命令行工具**，让开发者在终端中与 LLM 协作编辑代码。核心能力包括：多LLM支持、Git自动管理、多种代码编辑策略、仓库地图智能上下文、语音输入、Web界面等。

---

## 二、启动流程

```
python -m aider
  → aider/__main__.py          # 入口点
  → aider/main.py:main()       # 核心编排（1274行）
      ├─ get_parser()           # 参数解析
      ├─ load_dotenv_files()    # 加载.env
      ├─ setup_git()            # Git初始化
      ├─ register_models()      # 注册模型配置
      ├─ Model()                # 创建模型实例
      ├─ Coder.create()         # 创建Coder实例（工厂方法）
      └─ coder.run()            # 进入主聊天循环
```

## 六、对你改进 AI CLI 的10个关键学习点

### 1. **策略模式的Coder系统** ⭐⭐⭐
通过 `Coder.create(edit_format=...)` 工厂方法 + 子类继承，实现灵活切换编辑策略。建议你的CLI也采用这种模式，支持多种编辑方式。

### 2. **RepoMap智能上下文** ⭐⭐⭐
`repomap.py` 使用 tree-sitter 解析代码AST，只发送最相关的代码片段给LLM。这是控制token用量、提升编辑精度的核心。

### 3. **自动Git集成** ⭐⭐⭐
每次LLM编辑后自动commit，支持 `/undo` 撤销。让你的CLI可以安全地让AI修改代码。

### 4. **多模型协作（architect模式）** ⭐⭐
一个模型负责规划架构，另一个模型负责执行编辑。可以节省成本（用便宜模型执行简单任务）。

### 5. **litellm统一接口** ⭐⭐⭐
通过 `litellm` 库一个接口调用所有LLM，避免自己处理不同API差异。

### 6. **聊天历史摘要** ⭐⭐
`ChatSummary` 在对话过长时自动用weak_model摘要，防止超出上下文窗口。

### 7. **丰富的终端体验** ⭐⭐
`prompt_toolkit`（自动补全、历史、多行输入）+ `rich`（Markdown渲染、语法高亮）的组合，让CLI体验媲美Web。

### 8. **Prompt分层设计** ⭐⭐
system prompt / user reminder / examples 分层管理，每种编辑格式有独立的prompt文件。

### 9. **消息验证机制** ⭐
`sendchat.py` 确保消息角色交替、sanity check，防止API调用失败。

### 10. **配置文件灵活加载** ⭐
支持 `.aider.conf.yml` + `.env` + 命令行参数 + 环境变量，多层配置覆盖。
